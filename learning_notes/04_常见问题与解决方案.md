# 常见问题与解决方案

## 运行时错误

### 错误码 -1073741819 (0xC0000005)

**描述**：程序崩溃，退出代码 `-1073741819`

**原因分析**：
- OpenGL 上下文初始化失败
- 在错误的线程或上下文中调用 OpenGL 函数
- 多个 OpenGL Widget 共享上下文冲突

**解决方案**：

1. **确保在正确的上下文中初始化**
   ```python
   def initializeGL(self):
       self.makeCurrent()  # 获取当前 OpenGL 上下文
       live2d.init()
       live2d.glInit()
   ```

2. **添加共享 OpenGL 上下文属性**
   ```python
   # main.py
   from PyQt5.QtCore import Qt
   QApplication.setAttribute(Qt.AA_ShareOpenGLContexts, True)
   ```

3. **检查模型文件路径**
   ```python
   import os
   model_path = "./resources/models/hiyori.model3.json"
   if not os.path.exists(model_path):
       print(f"模型文件不存在: {model_path}")
   ```

### AttributeError: module 'live2d.v3' has no attribute 'glewInit'

**错误信息**：
```
AttributeError: module 'live2d.v3' has no attribute 'glewInit'. Did you mean: 'glInit'?
```

**原因**：函数名错误，应该是 `glInit` 不是 `glewInit`

**解决方案**：
```python
# 错误写法
live2d.glewInit()

# 正确写法
live2d.glInit()
```

---

### AttributeError: module 'live2d.v3' has no attribute 'setGLProperties'

**错误信息**：
```
AttributeError: module 'live2d.v3' has no attribute 'setGLProperties'
```

**原因**：新版本的 live2d-py 已经移除了 `setGLProperties()` 函数

**解决方案**：
```python
# 直接删除这行代码
def initializeGL(self):
    self.makeCurrent()
    live2d.init()
    live2d.glInit()
    # 不需要 setGLProperties()
```

---

### ImportError: cannot import name 'Qt' from 'PyQt5.QtGui'

**错误信息**：
```
ImportError: cannot import name 'Qt' from 'PyQt5.QtGui'
```

**原因**：`Qt` 类位于 `PyQt5.QtCore` 模块，而非 `PyQt5.QtGui`

**解决方案**：
```python
# 错误写法
from PyQt5.QtGui import Qt

# 正确写法
from PyQt5.QtCore import Qt
```

## 渲染问题

### 模型显示为空白/白色

**症状**：Live2D 模型渲染为空白或全白

**原因分析**：
1. OpenGL 上下文问题
2. 纹理加载失败
3. 着色器编译失败

**解决方案**：

1. **启用共享 OpenGL 上下文**
   ```python
   # main.py 开头
   from PyQt5.QtCore import Qt
   QApplication.setAttribute(Qt.AA_ShareOpenGLContexts, True)
   ```

2. **检查模型文件完整性**
   - 确认 `.moc3` 文件存在
   - 确认纹理文件（`.png`）存在且路径正确

3. **启用错误输出**
   ```python
   try:
       live2d_model.Draw()
   except Exception as e:
       print(f"渲染错误: {e}")
       import traceback
       traceback.print_exc()
   ```

---

### 渲染闪烁或不稳定

**症状**：窗口内容闪烁或不稳定

**解决方案**：
1. 减少帧率（从 60FPS 降到 30FPS）
   ```python
   self.startTimer(33)  # 约 30 FPS
   ```

2. 避免在 paintGL 中创建对象
   ```python
   def paintGL(self):
       # 在 initializeGL 中预创建的对象，不要在这里创建
       self.model.Draw()
   ```

---

### 窗口透明背景失效

**症状**：设置了透明背景但仍然不透明

**原因**：`QOpenGLWidget` 不支持 `WA_TranslucentBackground`

**解决方案**：
1. 使用 `QWidget` 作为主窗口
2. 在 `QWidget` 上叠加 `QOpenGLWidget`
3. 或者使用纹理贴图实现透明效果

```python
# 主窗口使用 QWidget，渲染区域使用 QOpenGLWidget
class DesktopPetWidget(QWidget):
    def __init__(self):
        super().__init__()
        self.setAttribute(Qt.WA_TranslucentBackground)
        self.live2d_widget = Live2DOpenGLWidget(self)
```

## 性能问题

### CPU 占用过高

**诊断**：程序 CPU 占用率达到 100%

**原因**：
- 帧率设置过高
- 每帧创建新对象
- 渲染循环有问题

**解决方案**：

1. **降低帧率**
   ```python
   # 从 60FPS 降到 30FPS
   self.startTimer(33)  # 33ms = 30FPS
   ```

2. **避免在渲染循环中创建对象**
   ```python
   # 在 initializeGL 中创建
   def initializeGL(self):
       self.some_pen = QPen()  # 不要这样做！

   # 正确做法
   def __init__(self):
       self.some_pen = QPen()  # 在这里创建
   ```

3. **检查 update() 调用**
   ```python
   # 确保没有多个定时器同时更新
   if self.update_timer.isActive():
       self.update_timer.stop()
   ```

---

### 内存占用持续增长

**症状**：程序运行时间越长，内存占用越大

**原因**：未释放之前的渲染资源

**解决方案**：
1. 复用模型实例，不要重复创建
2. 确保纹理只加载一次
3. 使用对象池管理频繁创建的对象

## 交互问题

### 鼠标位置不准确

**症状**：点击位置和实际响应位置不符

**原因**：坐标转换错误

**解决方案**：
```python
def mouseMoveEvent(self, event):
    # 使用 globalX()/globalY() 获取屏幕坐标
    # 然后转换为窗口坐标
    local_x = event.globalX() - self.x()
    local_y = event.globalY() - self.y()
    self.mouse_tracker.update_mouse_position(local_x, local_y)
```

---

### 点击事件无法触发动作

**症状**：点击模型没有响应

**原因**：未调用 `Touch()` 方法

**解决方案**：
```python
def mousePressEvent(self, event):
    if event.button() == Qt.LeftButton:
        # 触发动作
        self.live2d_model.StartRandomMotion("Tap", 2)
        # 或者使用 Touch
        self.live2d_model.Touch(event.pos().x(), event.pos().y())
```

---

### 拖动窗口时触发交互

**症状**：拖动窗口时意外触发点击动作

**原因**：未正确区分拖动和点击

**解决方案**：
```python
def mousePressEvent(self, event):
    if event.button() == Qt.LeftButton:
        self._drag_start = event.pos()
        self._dragging = True

def mouseReleaseEvent(self, event):
    if event.button() == Qt.LeftButton:
        # 只有移动距离很小才算点击
        if self._dragging and self._drag_start:
            dx = event.pos().x() - self._drag_start.x()
            dy = event.pos().y() - self._drag_start.y()
            if dx*dx + dy*dy < 100:  # 10像素范围内
                self._trigger_click()
        self._dragging = False
```

## 文件路径问题

### 模型加载失败

**症状**：`LoadModelJson()` 返回 `False`

**诊断步骤**：

1. **检查文件是否存在**
   ```python
   import os
   model_path = "./resources/models/hiyori.model3.json"
   print(f"文件存在: {os.path.exists(model_path)}")
   print(f"绝对路径: {os.path.abspath(model_path)}")
   ```

2. **检查相对路径**
   - 相对路径是相对于**当前工作目录**，不是文件所在目录
   - 建议使用绝对路径或确保工作目录正确

3. **检查 JSON 格式**
   ```python
   import json
   with open(model_path, 'r') as f:
       data = json.load(f)
       print("JSON 有效")
   ```

4. **检查文件引用**
   ```python
   # .model3.json 中的文件引用
   {
     "FileReferences": {
       "Moc": "hiyori.moc3",  # 相对于 .model3.json 的路径
       "Textures": ["hiyori.4096/texture_00.png"]
     }
   }
   ```

---

### 纹理文件加载失败

**症状**：模型显示为黑色或缺失部分

**原因**：纹理文件路径错误

**解决方案**：
- 确保纹理文件夹存在
- 检查文件名大小写（尤其 Linux 系统）
- 使用绝对路径测试

## 回退机制

### 设计原则

当 Live2D 渲染失败时，自动切换到模拟模式：

```python
class DesktopPetWidget(QOpenGLWidget):
    def __init__(self):
        super().__init__()
        self.live2d_model = None
        self._fallback_mode = False

    def initializeGL(self):
        self.makeCurrent()
        try:
            live2d.init()
            live2d.glInit()
            self.live2d_model = live2d.LAppModel()
            self.live2d_model.LoadModelJson(settings.MODEL_PATH)
            print("[Live2D] 真实渲染模式")
        except Exception as e:
            print(f"[Live2D] 初始化失败，使用模拟模式: {e}")
            self._fallback_mode = True

    def paintGL(self):
        if self.live2d_model and not self._fallback_mode:
            # 真实渲染
            live2d.clearBuffer()
            self.live2d_model.Update()
            self.live2d_model.Draw()
        else:
            # 模拟渲染
            painter = QPainter(self)
            self._fallback_render(painter)
```

## 快速诊断清单

1. **程序无法启动**
   - [ ] 检查 Python 版本 (3.8+)
   - [ ] 检查依赖安装 (`pip list`)
   - [ ] 检查模型文件路径

2. **渲染失败**
   - [ ] 启用 `AA_ShareOpenGLContexts`
   - [ ] 检查 `makeCurrent()` 调用
   - [ ] 启用异常捕获和日志

3. **性能问题**
   - [ ] 降低帧率到 30FPS
   - [ ] 避免在渲染循环中创建对象
   - [ ] 检查 CPU 占用率

4. **交互无响应**
   - [ ] 检查事件处理函数
   - [ ] 验证坐标转换
   - [ ] 测试 `Touch()` 和 `Drag()` 调用
