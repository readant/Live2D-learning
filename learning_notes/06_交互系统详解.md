3. **动作管理**
   - 动作优先级系统
   - 动作冷却时间
   - 动作回调机制
   - 动作状态追踪

## 点击交互逻辑

### 点击检测算法

```python
def _detect_hit_area(self, y: float) -> HitArea:
    """
    基于 Y 坐标判断命中区域

    Args:
        y: 点击的 Y 坐标（相对于窗口）

    Returns:
        HitArea: 命中区域枚举值
    """
    window_height = settings.WINDOW_HEIGHT
    relative_y = (y / window_height) * 100

    if relative_y < 40:
        return HitArea.HEAD  # 头部区域
    else:
        return HitArea.BODY  # 身体区域
```

### 点击类型判断

```python
def _handle_click(self, x: float, y: float) -> None:
    """处理点击事件"""
    current_time = time.time()
    hit_area = self._detect_hit_area(y)

    # 判断是否为连续点击
    if current_time - self._last_click_time > self._double_click_interval:
        self._click_count = 1
    else:
        self._click_count += 1

    self._last_click_time = current_time

    # 根据点击次数触发不同的定时器
    if self._click_count == 1:
        # 300ms 后确认是否为单击
        QTimer.singleShot(300, lambda: self._process_click_action(..., 1, ...))
    elif self._click_count == 2:
        # 500ms 后确认是否为双击
        QTimer.singleShot(500, lambda: self._process_click_action(..., 2, ...))
    elif self._click_count >= 3:
        # 立即触发三击
        self._process_click_action(..., 3, ...)
```

## 冷却系统

### 设计原理

冷却系统防止动作被频繁触发，提升用户体验：

```python
class InteractionController:
    def __init__(self):
        self._cooldowns = {
            "Tap": 1.0,      # 1秒冷却
            "Flick": 0.5,    # 0.5秒冷却
        }
        self._cooldown_timers = {}

    def _is_in_cooldown(self, motion_name: str) -> bool:
        """检查动作是否在冷却中"""
        if motion_name not in self._cooldowns:
            return False

        if motion_name not in self._cooldown_timers:
            return False

        elapsed = time.time() - self._cooldown_timers[motion_name]
        return elapsed < self._cooldowns[motion_name]

    def _set_cooldown(self, motion_name: str):
        """设置动作冷却"""
        if motion_name in self._cooldowns:
            self._cooldown_timers[motion_name] = time.time()
```

### 动态冷却调整


## 性能优化

### 1. 冷却时间设置

- 频繁触发的动作（如 `Tap`）设置较长冷却（0.5-1秒）
- 稀有动作（如 `FlickDown`）可以设置较短冷却

### 2. 避免频繁创建对象

```python
# 不好：在点击处理中创建对象
def _handle_click(self, x, y):
    hit_area = self._detect_hit_area(y)  # 每次创建

# 好：复用对象或使用枚举
def _detect_hit_area(self, y: float) -> HitArea:
    # 使用计算而非创建对象
```

### 3. 合理使用定时器

```python
# 使用单次定时器而非循环定时器
QTimer.singleShot(300, callback)  # 单次触发

# 避免：
# self.timer.timeout.connect(callback)  # 持续触发
```

## 常见问题排查

### 1. 动作不触发

**检查项**：
- [ ] 动作名称是否正确拼写
- [ ] 动作文件是否存在
- [ ] 是否在冷却中（查看日志）
- [ ] 优先级是否正确

**调试方法**：
```python
def _trigger_motion(self, motion_name: str):
    print(f"[调试] 尝试触发: {motion_name}")
    print(f"[调试] 冷却状态: {self._is_in_cooldown(motion_name)}")

    if self.live2d_model:
        print(f"[调试] 模型可用")
        # ... 触发代码 ...
```

### 2. 点击区域判断不准确

**原因**：窗口大小改变后区域划分未更新

**解决**：确保使用最新的 `settings.WINDOW_HEIGHT`

### 3. 双击被识别为两次单击

**原因**：点击间隔超过 `_double_click_interval`

**解决**：调整时间间隔或使用更好的点击检测算法
