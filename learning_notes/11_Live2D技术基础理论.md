## Live2D 核心技术原理

### 什么是 Live2D？

Live2D 是一种将静态图像转换为动态交互式角色的技术，通过以下方式实现：

- **多层图像叠加**：将角色拆分为多个可独立运动的部件
- **顶点变形**：通过移动图像顶点实现形变效果
- **透视变换**：模拟 3D 旋转效果
- **动态纹理**：实时渲染和混合

### 核心技术特点

| 特点 | 说明 | 应用场景 |
|------|------|----------|
| 2D 骨骼动画 | 类似 3D 骨骼的绑定系统 | 角色动作控制 |
| 关键帧动画 | 预设姿态之间的过渡 | 表情、动作切换 |
| 物理模拟 | 头发、衣服的物理运动 | 自然动态效果 |
| 参数驱动 | 通过数值控制动画状态 | 交互响应 |

### 与传统动画的区别

| 对比项 | 传统帧动画 | Live2D 技术 |
|--------|----------|-------------|
| 文件体积 | 大（每帧独立图像） | 小（部件复用） |
| 交互性 | 差 | 好 |
| 内存占用 | 高 | 低 |
| 动作平滑度 | 依赖帧数 | 可实时插值 |
| 角色定制 | 困难 | 灵活 |

## 模型设计原理

### 模型拆分原则

一个角色模型通常需要拆分为以下部件：

```
┌─────────────────────────────────┐
│           角色整体结构            │
├─────────────────────────────────┤
│  头发（前发）    ← 可独立运动      │
│  脸部            ← 表情变化区      │
│  眼睛            ← 视线关注        │
│  嘴巴            ← 口型变化        │
│  身体            ← 姿态控制        │
│  头发（后发）    ← 物理飘动        │
│  服装            ← 物理模拟        │
│  配饰            ← 独立部件        │
└─────────────────────────────────┘
```

### 部件拆分规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 运动独立性 | 相互独立运动的部件分开 | 左眼、右眼分开 |
| 形变需求 | 需要形变的区域单独拆分 | 嘴巴单独 |
| 物理特性 | 具有物理运动特性的单独拆分 | 头发、裙子 |
| 层级关系 | 考虑前后遮挡关系 | 前发、后发分开 |

### ArtMesh（艺术网格）

ArtMesh 是 Live2D 中的基本渲染单元：

- 每个 ArtMesh 包含一个图像区域
- 可以独立进行变换和形变
- 通过参数控制运动

```python
# ArtMesh 示例配置
artmesh_config = {
    "name": "hair_front",
    "src": "texture_00.png",
    "vertices": [...],  # 顶点坐标
    "uv": [...],        # 纹理坐标
    "indices": [...],    # 索引
    "parent": "head"     # 父级部件
}
```

### 拓扑结构

良好的拓扑结构确保平滑的形变效果：

```
好（三角剖分均匀）          差（三角剖分不均）
    ┌─┐                        ┌─┐
   /││\                       /│ │
  ╱ │ │ ╲                    ╱ │  │
 ╱  │ │  ╲                 ╱   │   ╲
┌───────┐                  ┌─────────┐
│       │                  │         │
└───────┘                  └─────────┘
```

## 骨骼绑定技术

### 骨骼系统概述

Live2D 采用类似 3D 动画的骨骼绑定系统：

| 概念 | 说明 |
|------|------|
| Bone（骨骼） | 控制部件运动的控制点 |
| Deformer（变形器） | 骨骼的容器，管理一组骨骼 |
| Parameter（参数） | 控制骨骼运动的数值 |

### 骨骼类型

#### 1. 旋转骨骼（Rotation Deformer）

用于控制部件的旋转运动：

```
        父骨骼
           │
           ▼
    ┌──────────┐
    │  旋转中心  │ ← 骨骼锚点
    └──────────┘
           │
      子骨骼/部件
           │
           ▼
        旋转角度 θ
```

#### 2. 移动骨骼（Translation Deformer）

用于控制部件的平移运动：

```
    初始位置          移动后
    ┌───┐            ┌───┐
    │ ■ │      →      │ ■ │
    └───┘            └───┘
     (x, y)         (x+dx, y+dy)
```

#### 3. 缩放骨骼（Scaling Deformer）

用于控制部件的缩放：

```
    1.0x              1.5x
    ┌───┐            ┌─────┐
    │ ■ │      →      │  ■  │
    └───┘            └─────┘
```

### 骨骼绑定流程

```
┌─────────────────────────────────────────┐
│           骨骼绑定流程                    │
├─────────────────────────────────────────┤
│  1. 创建 Deformer（变形器）              │
│         ↓                                │
│  2. 添加骨骼，设置骨骼类型                │
│         ↓                                │
│  3. 将 ArtMesh 绑定到骨骼                 │
│         ↓                                │
│  4. 设置骨骼参数范围                      │
│         ↓                                │
│  5. 调整绑定权重                         │
│         ↓                                │
│  6. 测试动画效果                         │
└─────────────────────────────────────────┘
```

### 绑定权重

每个 ArtMesh 可以受到多个骨骼的影响，权重决定影响程度：

```python
# 权重配置示例
bind_config = {
    "artmesh": "hair_front",
    "bones": [
        {"bone": "head_rotation", "weight": 1.0},   # 完全跟随头部
        {"bone": "hair_physics", "weight": 0.3}   # 30% 物理影响
    ]
}
```

### 骨骼命名规范

| 规范 | 示例 | 说明 |
|------|------|------|
| 部位_类型 | Head_Rot | 头部旋转骨骼 |
| 方向_序号 | Arm_L_01 | 左臂骨骼01 |
| 功能_位置 | Eye_Target | 眼睛目标点 |

## 参数系统详解

### 参数类型

| 参数类型 | 参数ID前缀 | 说明 | 取值范围 |
|----------|-----------|------|----------|
| 角度 | ParamAngle | 旋转角度 | 通常 -30 ~ 30 |
| 位置 | ParamCenter | 平移位置 | 通常 -1 ~ 1 |
| 缩放 | ParamScale | 缩放比例 | 通常 0.5 ~ 1.5 |
| 可见性 | ParamVisibility | 显示/隐藏 | 0 或 1 |

### 常用参数列表

| 参数名 | 中文说明 | 取值范围 | 常见用途 |
|--------|----------|----------|----------|
| ParamAngleX | 横向旋转 | -30 ~ 30 | 头部左右看 |
| ParamAngleY | 纵向旋转 | -30 ~ 30 | 头上仰下看 |
| ParamAngleZ | 深度旋转 | -30 ~ 30 | 头部歪斜 |
| ParamEyeBallX | 眼球横向 | -1 ~ 1 | 视线左右 |
| ParamEyeBallY | 眼球纵向 | -1 ~ 1 | 视线上下 |
| ParamEyeLOpen | 左眼睁开度 | 0 ~ 1 | 眨眼 |
| ParamEyeROpen | 右眼睁开度 | 0 ~ 1 | 眨眼 |
| ParamMouthForm | 嘴巴形状 | -1 ~ 1 | 张嘴闭嘴 |
| ParamMouthOpenY | 嘴巴张开度 | 0 ~ 1 | 说话口型 |
| ParamBreath | 呼吸 | 0 ~ 1 | 呼吸起伏 |
| ParamHairFluffy | 头发蓬松度 | 0 ~ 1 | 头发膨胀 |

### 参数关键帧

参数值可以设置关键帧，形成动画：

```json
{
    "Parameter": "ParamAngleX",
    "KeyValues": [
        {"time": 0.0, "value": 0.0, "innter": "linear"},
        {"time": 0.5, "value": 30.0, "innter": "easeInOut"},
        {"time": 1.0, "value": 0.0, "innter": "linear"}
    ]
}
```

### 插值类型

| 插值类型 | 效果 | 适用场景 |
|----------|------|----------|
| linear | 线性均匀 | 平滑动作 |
| easeIn | 先慢后快 | 进入效果 |
| easeOut | 先快后慢 | 退出效果 |
| easeInOut | 慢快慢 | 自然过渡 |
| overshoot | 超过目标 | 弹性效果 |

## 模型文件结构

### Cubism 4.0 模型文件

```
model_name/
├── model_name.model3.json    # 主配置文件
├── model_name.moc3           # 模型数据文件（二进制）
├── model_name.2048/          # 纹理图集
│   └── texture_00.png
├── model_name.physics3.json  # 物理演算配置
├── model_name.pose3.json    # 姿态配置
├── model_name.motion3.json   # 动作文件（多个）
│   ├── idle.motion3.json
│   ├── tap.motion3.json
│   └── ...
└── model_name.expression.json # 表情文件（多个）
    ├── happy.expression.json
    └── sad.expression.json
```

### model3.json 结构

```json
{
    "Version": 3,
    "FileReferences": {
        "Moc": "Hiyori.moc3",
        "Textures": ["Hiyori.2048/texture_00.png"],
        "Physics": "Hiyori.physics3.json",
        "Motions": {
            "Idle": ["motion/Hiyori_m01.motion3.json"],
            "Tap": ["motion/Hiyori_m02.motion3.json"]
        }
    },
    "Groups": [
        {
            "Target": "Parameter",
            "Name": "EyeBlink",
            "Ids": ["ParamEyeLOpen", "ParamEyeROpen"]
        }
    ]
}
```

### 纹理图集（Texture Atlas）

纹理图集将多个部件图像合并，减少绘制调用：

```
┌─────────────────────────┐
│  皮肤   眼睛   嘴巴      │
├─────────────────────────┤
│  头发   服装   配饰      │
└─────────────────────────┘
     texture_00.png
```

### 练习 1：理解部件拆分

**目标**：理解角色模型的部件拆分原则

**步骤**：
1. 观察项目中的 Hiyori 模型

**扩展**：

- 总结拆分规律

### 练习 2：参数调试

**目标**：熟悉 Live2D 参数系统

**步骤**：
1. 打开 `src/live2d/model.py`
2. 找到 `set_parameter` 相关方法
3. 修改参数值，观察模型变化

**模板代码**：
```python
# 设置头部旋转
model.set_parameter("ParamAngleX", 30.0)  # 头部右转30度
model.set_parameter("ParamAngleY", -15.0) # 头部下看15度

# 设置眼球注视
model.set_parameter("ParamEyeBallX", 0.5)  # 眼球右看
model.set_parameter("ParamEyeBallY", 0.3) # 眼球上看
```

### 练习 3：骨骼绑定分析

**目标**：理解骨骼和变形器的关系

**步骤**：
1. 分析模型的 Deformer 结构
2. 追踪参数到骨骼的运动传递路径
3. 绘制骨骼层级图
