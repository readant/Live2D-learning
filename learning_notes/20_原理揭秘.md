## Live2D 核心原理

### 1.1 Live2D 工作原理

Live2D 的核心思想是 **2D 图像的 3D 变形**：

```
┌─────────────────────────────────────────────────────┐
│              Live2D 渲染流程                       │
├─────────────────────────────────────────────────────┤
│                                                   │
│  原始图像 ───→ 网格划分 ───→ 骨骼绑定 ───→ 参数驱动  │
│       │               │            │            │   │
│       ↓               ↓            ↓            ↓   │
│   纹理图片        顶点网格        变形权重      动画   │
│                                                   │
└─────────────────────────────────────────────────────┘
```

**关键技术点**：
- **网格变形**：通过修改顶点位置实现平滑变形
- **骨骼系统**：层级化骨骼结构，父骨骼带动子骨骼
- **权重映射**：每个顶点受多个骨骼影响的权重

### 1.2 MOC3 文件格式解析

MOC3（Model Core 3）是 Live2D 的核心模型文件：

```
MOC3 文件结构
├── Header（文件头）
│   ├── Magic Number（魔数）
│   ├── Version（版本号）
│   └── File Size（文件大小）
├── Model Data（模型数据）
│   ├── Vertex Count（顶点数）
│   ├── Index Count（索引数）
│   ├── Texture Info（纹理信息）
│   └── Mesh Data（网格数据）
├── Bone Data（骨骼数据）
│   ├── Bone Count（骨骼数）
│   ├── Parent Relationships（父子关系）
│   └── Transform Data（变换数据）
└── Parameter Data（参数数据）
    ├── Parameter Count（参数数）
    ├── Parameter Names（参数名）
    └── Parameter Ranges（参数范围）
```

### 1.3 骨骼绑定原理

骨骼绑定是将网格顶点与骨骼关联的过程：

```python
# 简化的骨骼绑定计算
def calculate_vertex_position(vertex, bones, weights):
    """
    计算顶点最终位置
    :param vertex: 原始顶点位置
    :param bones: 骨骼列表
    :param weights: 权重列表（与骨骼对应）
    :return: 变换后的顶点位置
    """
    result = np.zeros(2)
    
    for bone, weight in zip(bones, weights):
        # 获取骨骼变换矩阵
        transform = bone.get_transform_matrix()
        
        # 应用变换
        transformed = np.dot(transform, np.array([vertex[0], vertex[1], 1]))
        
        # 加权求和
        result += transformed[:2] * weight
    
    return result
```

**权重约束**：所有影响同一顶点的骨骼权重之和必须等于 1

## 面部捕捉原理

### 2.1 MediaPipe Face Mesh 原理

Face Mesh 使用 **深度学习模型** 进行人脸关键点检测：

```
┌─────────────────────────────────────────────────────┐
│           MediaPipe Face Mesh 架构                  │
├─────────────────────────────────────────────────────┤
│                                                   │
│  输入图像 ───→ 人脸检测 ───→ 关键点回归 ───→ 输出   │
│       │               │            │            │   │
│       ↓               ↓            ↓            ↓   │
│    RGB图像        BlazeFace     ResNet18    468点   │
│                   (人脸检测)   (关键点)   3D坐标    │
│                                                   │
└─────────────────────────────────────────────────────┘
```

**技术细节**：
- 使用 **BlazeFace** 进行人脸检测和裁剪
- 使用 **ResNet18** 作为骨干网络进行关键点回归
- 输出 **468 个 3D 关键点**（包含深度信息）

### 2.2 PnP 算法原理

PnP（Perspective-n-Point）算法用于从 2D 图像点估计 3D 姿态：

```python
def pnp_algorithm(image_points, model_points, camera_matrix):
    """
    PnP 算法核心思想
    :param image_points: 2D 图像点
    :param model_points: 3D 模型点（已知）
    :param camera_matrix: 相机内参
    :return: 旋转向量、平移向量
    """
    # 最小化重投影误差
    # min ||image_points - project(model_points, R, t)||^2
    
    # 常用方法：
    # 1. SOLVEPNP_ITERATIVE - Levenberg-Marquardt 迭代
    # 2. SOLVEPNP_P3P - 基于几何方法（需至少4个点）
    # 3. SOLVEPNP_EPNP - 高效的线性方法
    
    success, rotation_vec, translation_vec = cv2.solvePnP(
        model_points, image_points, camera_matrix, None,
        flags=cv2.SOLVEPNP_ITERATIVE
    )
    
    return rotation_vec, translation_vec
```

**数学原理**：
- 利用 **透视投影模型**
- 最小化 **重投影误差**（Reprojection Error）
- 使用 **Levenberg-Marquardt** 优化算法

### 2.3 欧拉角与旋转矩阵

姿态表示的三种方式：

| 表示方式 | 优点 | 缺点 |
|----------|------|------|
| **欧拉角** | 直观，易于理解 | 万向锁问题 |
| **旋转矩阵** | 无万向锁，易于组合 | 9个参数，冗余 |
| **四元数** | 紧凑，无万向锁 | 不易直观理解 |

```python
def rotation_matrix_to_euler(R):
    """
    旋转矩阵转欧拉角
    :param R: 3x3 旋转矩阵
    :return: (pitch, yaw, roll)
    """
    # 提取欧拉角
    sy = np.sqrt(R[0, 0]**2 + R[1, 0]**2)
    
    if sy > 1e-6:
        # 正常情况
        roll = np.arctan2(R[2, 1], R[2, 2])
        pitch = np.arctan2(-R[2, 0], sy)
        yaw = np.arctan2(R[1, 0], R[0, 0])
    else:
        # 奇异情况（万向锁）
        roll = np.arctan2(-R[1, 2], R[1, 1])
        pitch = np.arctan2(-R[2, 0], sy)
        yaw = 0.0
    
    return np.array([pitch, yaw, roll])
```

## 动画系统原理

### 3.1 关键帧动画原理

关键帧动画通过插值实现平滑过渡：

```python
def linear_interpolation(keyframes, time):
    """
    线性插值
    :param keyframes: 关键帧列表 [(time1, value1), (time2, value2), ...]
    :param time: 当前时间
    :return: 插值后的值
    """
    # 找到当前时间所在的关键帧区间
    for i in range(len(keyframes) - 1):
        t0, v0 = keyframes[i]
        t1, v1 = keyframes[i+1]
        
        if t0 <= time <= t1:
            # 计算插值系数
            alpha = (time - t0) / (t1 - t0)
            return v0 + alpha * (v1 - v0)
    
    return keyframes[-1][1]  # 返回最后一个值
```

### 3.2 曲线插值类型

| 插值类型 | 公式 | 效果 |
|----------|------|------|
| **Linear** | y = x | 匀速运动 |
| **Ease In** | y = x² | 慢启动 |
| **Ease Out** | y = 1 - (1-x)² | 慢结束 |
| **Ease In Out** | y = 2x² (x<0.5), y=1-2(1-x)² | 慢启动慢结束 |

### 3.3 动画混合原理

多个动画可以叠加混合：

```python
def blend_animations(animations, weights):
    """
    动画混合
    :param animations: 动画列表
    :param weights: 权重列表（和为1）
    :return: 混合后的参数值
    """
    result = {}
    
    # 遍历所有参数
    for param_name in animations[0].keys():
        value = 0
        for anim, weight in zip(animations, weights):
            value += anim[param_name] * weight
        result[param_name] = value
    
    return result
```

## 物理引擎原理

### 4.1 弹簧-阻尼系统

物理引擎基于 **弹簧-阻尼模型**：

```python
def spring_damper_update(spring, delta_time):
    """
    更新弹簧状态
    :param spring: 弹簧对象
    :param delta_time: 时间步长
    """
    # Hooke 定律：F = -k * x - b * v
    # k: 弹簧系数，b: 阻尼系数
    
    # 计算位移
    displacement = spring.current_pos - spring.rest_pos
    
    # 计算力
    spring_force = -spring.stiffness * displacement
    damping_force = -spring.damping * spring.velocity
    
    total_force = spring_force + damping_force
    
    # 更新加速度（F = ma）
    acceleration = total_force / spring.mass
    
    # 更新速度和位置（Verlet 积分）
    spring.velocity += acceleration * delta_time
    spring.current_pos += spring.velocity * delta_time
```

## 参数映射原理

### 5.1 参数映射流程

将面部捕捉数据映射到 Live2D 参数：

```python
def map_parameter(input_value, input_range, output_range, curve_type='linear'):
    """
    参数映射
    :param input_value: 输入值
    :param input_range: 输入范围 (min, max)
    :param output_range: 输出范围 (min, max)
    :param curve_type: 曲线类型
    :return: 映射后的值
    """
    # 1. 裁剪到输入范围
    input_value = np.clip(input_value, *input_range)
    
    # 2. 归一化到 [0, 1]
    normalized = (input_value - input_range[0]) / (input_range[1] - input_range[0])
    
    # 3. 应用曲线
    normalized = apply_curve(normalized, curve_type)
    
    # 4. 映射到输出范围
    output = output_range[0] + normalized * (output_range[1] - output_range[0])
    
    return output
```

### 5.2 曲线应用函数

```python
def apply_curve(value, curve_type):
    """
    应用映射曲线
    :param value: 归一化值 [0, 1]
    :param curve_type: 曲线类型
    :return: 处理后的值
    """
    if curve_type == 'linear':
        return value
    elif curve_type == 'ease_in':
        return value ** 2
    elif curve_type == 'ease_out':
        return 1 - (1 - value) ** 2
    elif curve_type == 'ease_in_out':
        if value < 0.5:
            return 2 * value ** 2
        else:
            return 1 - (-2 * value + 2) ** 2 / 2
    else:
        return value
```

## 性能优化原理

### 6.1 平滑滤波原理

#### 6.1.1 EWMA（指数加权移动平均）

```python
def ewma_filter(new_value, prev_value, alpha=0.3):
    """
    EWMA 滤波
    :param new_value: 新值
    :param prev_value: 上一次滤波值
    :param alpha: 平滑系数 (0-1)，越小越平滑
    :return: 滤波后的值
    """
    # S_t = alpha * X_t + (1 - alpha) * S_{t-1}
    return alpha * new_value + (1 - alpha) * prev_value
```

**时间复杂度**: O(1)
**空间复杂度**: O(1)

#### 6.1.2 卡尔曼滤波

卡尔曼滤波是一种 **递归贝叶斯滤波器**：

```python
def kalman_filter(z, x_prev, p_prev, q=0.01, r=0.1):
    """
    简化卡尔曼滤波
    :param z: 测量值
    :param x_prev: 上一次估计值
    :param p_prev: 上一次估计误差协方差
    :param q: 过程噪声
    :param r: 测量噪声
    :return: (新估计值, 新误差协方差)
    """
    # 预测步骤
    x_pred = x_prev
    p_pred = p_prev + q
    
    # 更新步骤
    k = p_pred / (p_pred + r)  # 卡尔曼增益
    x = x_pred + k * (z - x_pred)
    p = (1 - k) * p_pred
    
    return x, p
```

### 6.2 帧率优化策略

| 优化方法 | 原理 | 效果 |
|----------|------|------|
| **降低分辨率** | 减少图像处理像素数 | 显著提升 FPS |
| **跳过帧** | 每隔N帧处理一次 | 平衡性能和流畅度 |
| **多线程** | 并行处理检测和渲染 | 充分利用多核 |
| **GPU加速** | 使用GPU进行图像处理 | 大幅提升性能 |
