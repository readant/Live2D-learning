## 动画基础概念

### 帧率与时间

| 概念 | 说明 | 常用值 |
|------|------|--------|
| FPS | 每秒帧数 | 30 / 60 |
| 帧时长 | 每帧持续时间 | 0.033s / 0.017s |
| 动画时长 | 完整动画持续时间 | 根据动作而定 |

### 动画类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 循环动画 | 持续重复播放 | 待机呼吸、眨眼 |
| 触发动画 | 特定条件触发 | 点击挥手、滑动 |
| 过渡动画 | 状态切换动画 | 表情切换 |
| 连续动画 | 多段动作组合 | 跳舞、表演 |

### 动画曲线

动画曲线控制参数值随时间的变化：

```
值
 ↑
 │    ╭──╮
 │   ╱    ╲            ← 缓入缓出
 │  ╱      ╲
 │ ╱        ╲
 │╱          ╲──────
 └────────────────→ 时间
```

## 动作文件结构

### motion3.json 结构

```json
{
    "Version": 3,
    "Meta": {
        "Duration": 2.567,
        "Loop": true,
        "AreBridgesExist": false,
        "Fps": 30.0,
        "CurveCount": 5
    },
    "Curves": [
        {
            "Target": "Parameter",
            "Id": "ParamAngleX",
            "Segments": [0.0, 0.0, 0.5, 30.0, 1.0, 0.0, 2.567, "linear"]
        },
        {
            "Target": "Parameter",
            "Id": "ParamAngleY",
            "Segments": [0.0, 0.0, 1.0, 0.0]
        }
    ]
}
```

### 曲线（Curve）

每个 Curve 控制一个参数的时间变化：

```python
curve = {
    "Target": "Parameter",  # 参数目标
    "Id": "ParamAngleX",    # 参数ID
    "Segments": [...]        # 时间-值曲线
}
```

### 段（Segment）

Segments 是连续的时间-值点列表：

| 类型 | 格式 | 说明 |
|------|------|------|
| 线性 | `[t0, v0, t1, v1, ...]` | 线性插值 |
| 贝塞尔 | `[t0, v0, t1, v1, vt1, vt2, vt3, v3]` | 贝塞尔曲线插值 |

## 关键帧动画

### 关键帧原理

关键帧动画通过定义关键时间点的参数值，自动插值生成中间帧：

```
关键帧1          关键帧2          关键帧3
   │                │                │
   ▼                ▼                ▼
  t=0s            t=1s             t=2s
  angle=0°    →   angle=30°    →   angle=0°
                ↑
            中间帧自动生成
            t=0.5s
            angle=15°
```

### 关键帧设置

```python
# 关键帧数据示例
keyframes = [
    {"time": 0.0, "value": 0.0, "easing": "linear"},
    {"time": 0.5, "value": 30.0, "easing": "easeInOut"},
    {"time": 1.0, "value": 0.0, "easing": "linear"}
]
```

### 插值算法

#### 线性插值 (Linear)

```python
def lerp(v0, v1, t):
    return v0 + (v1 - v0) * t

# 示例：0 + (30 - 0) * 0.5 = 15
```

#### 贝塞尔插值 (Bezier)

```python
def bezier(p0, p1, p2, p3, t):
    """三次贝塞尔曲线"""
    return (1-t)**3 * p0 + 3*(1-t)**2 * t * p1 + 3*(1-t) * t**2 * p2 + t**3 * p3
```

## 动作分组与循环

### 动作分组

Live2D 支持将多个动作分组管理：

```json
{
    "Groups": [
        {
            "Target": "Parameter",
            "Name": "LipSync",
            "Ids": ["ParamMouthOpenY"]
        },
        {
            "Target": "Parameter",
            "Name": "EyeBlink",
            "Ids": ["ParamEyeLOpen", "ParamEyeROpen"]
        }
    ]
}
```

### 循环动作

待机动画通常是循环播放的：

```python
# 循环播放待机动画
class IdleMotion:
    def __init__(self):
        self.duration = 4.0  # 循环周期4秒
        self.is_loop = True

    def play(self):
        self.model.start_motion("Idle", loop=self.is_loop)

    def update(self, delta_time):
        self.elapsed += delta_time
        if self.elapsed >= self.duration:
            self.elapsed = 0.0  # 重置循环
```

### 非循环动作

触发动作通常只播放一次：

```python
# 触发一次的动作
class TapMotion:
    def __init__(self):
        self.duration = 2.0
        self.is_loop = False
        self.play_count = 0

    def play(self):
        self.model.start_motion("TapBody", loop=False)
        self.play_count += 1

    def is_finished(self):
        return self.play_count > 0 and self.elapsed >= self.duration
```

## 动作优先级

### 优先级系统

当多个动作同时播放时，优先级决定最终效果：

| 优先级 | 动作类型 | 说明 |
|--------|----------|------|
| 0 | 待机动画 | 最低优先级 |
| 1 | 循环动作 | 背景动作 |
| 2 | 交互动作 | 用户触发 |
| 3 | 特殊动作 | 重要事件 |

### 优先级实现

```python
class MotionPriority:
    IDLE = 0
    BACKGROUND = 1
    INTERACTIVE = 2
    SPECIAL = 3

class MotionController:
    def __init__(self):
        self.active_motions = {}

    def play_motion(self, name, priority, motion_data):
        # 检查优先级
        if name in self.active_motions:
            current_priority = self.active_motions[name]['priority']
            if current_priority > priority:
                return False  # 拒绝低优先级替换

        # 播放动作
        self.active_motions[name] = {
            'priority': priority,
            'motion': motion_data,
            'start_time': time.time()
        }
        return True
```

### 动作中断

高优先级动作可以中断低优先级动作：

```python
def interrupt_motion(self, new_motion_name, priority):
    # 停止所有低优先级动作
    for name, data in list(self.active_motions.items()):
        if data['priority'] < priority:
            self.stop_motion(name)
            del self.active_motions[name]

    # 播放新动作
    self.play_motion(new_motion_name, priority, ...)
```

### 呼吸动画

呼吸是循环动画的典型示例：

```python
class BreathAnimation:
    def __init__(self):
        self.phase = 0.0
        self.speed = 0.5  # 呼吸周期倍率
        self.amplitude = 0.05  # 呼吸幅度

    def update(self, delta_time):
        self.phase += delta_time * self.speed
        if self.phase > 2 * math.pi:
            self.phase -= 2 * math.pi

        # 计算呼吸参数值
        breath_value = (math.sin(self.phase) + 1) / 2 * self.amplitude
        self.model.set_parameter("ParamBreath", breath_value)
```

### 眨眼动画

眨眼是随机触发的循环动画：

```python
class BlinkAnimation:
    def __init__(self):
        self.blink_interval = 3.0  # 平均眨眼间隔
        self.blink_timer = random.uniform(2.0, 4.0)
        self.blink_duration = 0.1  # 眨眼完成时间
        self.is_blinking = False
        self.blink_progress = 0.0

    def update(self, delta_time):
        if not self.is_blinking:
            self.blink_timer -= delta_time
            if self.blink_timer <= 0:
                self.is_blinking = True
                self.blink_progress = 0.0
                self.blink_timer = random.uniform(2.0, 4.0)
        else:
            self.blink_progress += delta_time / self.blink_duration
            if self.blink_progress >= 1.0:
                self.is_blinking = False
                self.blink_progress = 0.0

        # 计算眨眼参数
        eye_open = 1.0
        if self.is_blinking:
            eye_open = 1.0 - math.sin(self.blink_progress * math.pi)

        self.model.set_parameter("ParamEyeLOpen", eye_open)
        self.model.set_parameter("ParamEyeROpen", eye_open)
```

## 动作制作流程

### 1. 规划动画

```
┌─────────────────────────────────────┐
│          动画规划阶段                │
├─────────────────────────────────────┤
│  • 确定动画类型（循环/触发）         │
│  • 确定动画时长                      │
│  • 列出需要变化的参数                │
│  • 设定关键帧位置                    │
└─────────────────────────────────────┘
```

### 2. 制作关键帧

在 Cubism Editor 中：

1. 选择要制作动画的参数
2. 在时间轴上添加关键帧
3. 调整关键帧的参数值
4. 设置关键帧之间的插值曲线

### 3. 导出动作文件

1. 预览动画效果
2. 调整时间轴
3. 导出为 motion3.json

### 4. 集成到项目

1. 将 motion3.json 放入 resources/models/motion/
2. 在代码中引用动作名称
3. 测试动作播放效果

## 常见问题

### Q1: 动作播放不连贯

**原因**：关键帧间隔不均匀或插值曲线不当

**解决**：
- 调整关键帧位置，使变化平滑
- 选择合适的插值曲线
- 增加中间关键帧

### Q2: 动作播放太快/太慢

**原因**：FPS 设置不正确或动画时长不合适

**解决**：
- 检查 motion3.json 中的 FPS 设置
- 调整动画时长
- 修改代码中的播放速度

### Q3: 多个动作冲突

**原因**：优先级设置不当

**解决**：
- 合理设置动作优先级
- 实现动作队列系统
- 添加动作过渡效果
